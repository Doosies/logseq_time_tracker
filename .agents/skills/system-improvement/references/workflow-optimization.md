# 워크플로우 최적화

## 최적화 전략

### 1. 불필요한 단계 제거
```
Before: 기획 → 구현 → QA → 문서화
After: 구현 → QA (간단한 버그)
```

### 2. 병렬화
```
Before: 구현 → QA → 문서화 (순차)
After: 구현 → (QA + 문서화) (병렬)
```

### 3. 조건부 단계
```
간단한 기능: 문서화 생략
복잡한 기능: 기획 필수
```

## 워크플로우 변형

### Feature 워크플로우 변형

**Light Feature** (10분 이내):
```
구현 → QA (빠른 테스트) → 완료
```

**Standard Feature** (30분):
```
기획 → 구현 → QA → 문서화
```

**Complex Feature** (1시간+):
```
워크플로우 관리자 호출
→ 태스크 분해
→ 병렬 실행
```
