## 목표

Cursor IDE에서 실제 코드 리뷰 시나리오로 MCP 서버를 테스트한다. 검색 결과가 실제로 유용한 코드 리뷰 컨텍스트를 제공하는지 수동 평가한다.

## 검증 시나리오

### 시나리오 1: 코드 리뷰에서 회귀 위험 감지

```
1. 대상 리포에서 실제 코드 변경 diff 선택
2. Cursor에서 코드 리뷰 컨텍스트에 cr-rag-mcp 연동
3. LLM이 search_review_context 자동 호출
4. 반환된 과거 히스토리가 현재 변경과 관련 있는지 평가
```

**성공 기준**: 관련 과거 변경 1개 이상 반환, 회귀 위험 언급

### 시나리오 2: 인제스트 워크플로우

```
1. ingest_commits(mode: incremental) 호출
2. 새 커밋이 정상적으로 인덱싱되는지 확인
3. 인덱싱 후 즉시 검색에 반영되는지 확인
```

**성공 기준**: 새 커밋 인제스트 → 검색 반영 1분 이내

### 시나리오 3: 이유 보강

```
1. reason_known: false인 커밋 확인
2. supplement_reason으로 이유 보강
3. 보강 후 검색 결과에 반영되는지 확인
```

**성공 기준**: 보강된 이유가 검색 결과 summary에 포함

### 시나리오 4: 콜드 스타트

```
1. 빈 데이터로 MCP 서버 시작
2. search_review_context 호출
3. 안내 메시지 반환 확인
4. ingest_commits(mode: bulk) 호출
5. 인제스트 후 검색 동작 확인
```

**성공 기준**: 콜드 스타트 안내 → 벌크 인제스트 → 검색 동작 전체 흐름 정상
