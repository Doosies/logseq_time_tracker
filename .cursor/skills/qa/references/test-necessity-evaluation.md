# 테스트 필요성 판단 가이드

코드 변경 시 테스트 작성 여부를 판단하는 기준과 절차입니다. **강제가 아닌 선택적 판단**이며, 미작성 시에는 반드시 근거를 기록합니다.

---

## 판단 기준 체크리스트

### 1. 회귀 위험

| 수준 | 설명 | 예시 |
|------|------|------|
| 낮음 | 기존 동작 변경 없음 | 설정값 변경, 주석 추가, 타입만 수정 |
| 중간 | 로직 일부 수정 | 조건문 추가, 헬퍼 함수 변경 |
| 높음 | 핵심 비즈니스 로직 변경 | 계산식, 인증 흐름, 데이터 변환 |

### 2. 복잡도

| 수준 | 설명 | 예시 |
|------|------|------|
| 낮음 | 10줄 이하, 단순 | getter 반환, 상수 export |
| 중간 | 분기 2~3개, 50줄 이내 | 유효성 검사, 포맷 변환 |
| 높음 | 분기 다수, 알고리즘 포함 | 정렬·필터·집계 로직 |

### 3. 외부 의존성

| 수준 | 설명 | 예시 |
|------|------|------|
| 낮음 | 순수 함수, 의존성 없음 | 유틸 함수 |
| 중간 | API·DB 등 mock 가능 | 서비스 레이어 |
| 높음 | 여러 외부 의존, 통합 필요 | E2E 시나리오 |

### 4. UI 상호작용

| 수준 | 설명 | 예시 |
|------|------|------|
| 낮음 | 정적 렌더링만 | 단순 표시 컴포넌트 |
| 중간 | 이벤트 핸들러 1~2개 | 버튼 클릭, input 변경 |
| 높음 | 폼, 비동기, 상태 복잡 | 다단계 폼, 실시간 검증 |

---

## 판단 결과 예시

### 테스트 미작성 예시

| 상황 | 근거 |
|------|------|
| 단순 prop 이름 변경 | 기존 동작 동일, 회귀 위험 거의 없음 |
| 타입 정의만 추가 | 런타임 동작 변경 없음 |
| 주석·포맷 수정 | 로직 변경 없음 |
| 5줄 이하 getter 수정 | 복잡도·위험 모두 낮음 |
| 스타일/CSS만 변경 | 비즈니스 로직 무관 |

### 테스트 작성 예시

| 상황 | 이유 |
|------|------|
| 계산·변환 로직 추가 | 회귀 위험 높음, 경계값 검증 필요 |
| API 호출 로직 변경 | mock로 응답 검증 필요 |
| 폼 유효성 검사 수정 | 엣지 케이스 회귀 위험 |
| 인증·권한 관련 변경 | 보안·비즈니스 중요도 높음 |
| 기존 테스트 없는 핵심 모듈 수정 | 회귀 방지 필요 |

## 작성 시 최소 세트 가이드

테스트를 작성하기로 한 경우, **최소 유효 세트**를 권장합니다.

### 최소 구성

1. **핵심 성공 경로 1개**
   - 정상 입력에서 기대값 검증
   - 예: `it('정상 입력 시 올바른 결과를 반환해야 함')`

2. **실패/엣지 케이스 1개 이상**
   - 빈 입력, null, 잘못된 타입, 경계값 등
   - 예: `it('빈 배열일 때 ValidationError를 던져야 함')`

### 파일 위치

- `__tests__/모듈명.test.ts` 또는
- `모듈명.test.ts` (모듈 옆 배치)

### 명명 규칙

- `describe('함수/컴포넌트명')`
- `it('한글 설명')` (프로젝트 컨벤션)

## 보고 문구 템플릿

### 테스트 미작성 시 보고 문구

```markdown
## 테스트 판단: 미작성

- **점수표 합계**: [N]점
- **근거**: [예: 단순 prop 전달 변경, 로직 변경 없음, 회귀 위험 낮음]
- **검토한 대안**: [예: 최소 1개 테스트 작성 고려 → 변경 범위 대비 비용 대비 효과 낮아 생략]
```

### 테스트 작성 시 보고 문구

```markdown
## 테스트 판단: 작성 완료

- **점수표 합계**: [N]점
- **작성 파일**: `[경로]`
- **테스트 개수**: 성공 1, 실패/엣지 [N]
- **실행 결과**: [통과/실패]
- **커버리지**: [해당 시 기입]
```
