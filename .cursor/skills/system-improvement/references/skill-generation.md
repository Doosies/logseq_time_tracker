# Skill 생성 및 수정

## 새 Skill 생성 조건

### 패턴 감지
동일한 작업이 5회 이상 반복:
```
QA 에이전트가 매번 커버리지를 수동 파싱
→ coverage-automation.md 생성
```

### 지식 부족
에이전트가 특정 영역에서 반복적으로 실패:
```
API 보안 설정 실수 3회
→ api-security-checklist.md 생성
```

## 범용 vs 프로젝트 특화 판단 기준

스킬을 생성하거나 수정할 때, 내용이 **범용 패턴**인지 **프로젝트 특화 지식**인지 판단합니다:

| 기준 | 도메인 스킬 (`*/references/`) | 프로젝트 지식 (`project-knowledge/references/`) |
|------|------|------|
| 재사용성 | 다른 프로젝트에서도 적용 가능 | 이 프로젝트에서만 유효 |
| 내용 | 패턴, 원칙, 체크리스트 | 구체적 함수명, 변수명, 초기화 순서 |
| 예시 | "FOUC 방지를 위해 2단계 초기화 패턴 적용" | "`initializeTheme()`이 2단계" |
| 변경 빈도 | 거의 변하지 않음 | 코드 변경 시 함께 업데이트 |

**판단 질문**: "이 내용에서 함수명/변수명을 제거해도 의미가 통하는가?"
- YES → 도메인 스킬
- NO → 프로젝트 지식

## Skill 수정

### 불명확한 지침
에이전트가 지침을 잘못 이해:
```
"테스트 작성" → 단위만? 통합도?
→ 명확하게 수정
```

### 최신화
새로운 best practice:
```
React 18 업데이트
→ 관련 Skill 업데이트
```
