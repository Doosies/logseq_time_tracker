# Rule 최적화

## 최적화 대상

### 1. 품질 게이트 기준 조정
현재 문제:
- QA 단계 재시도 많음
- 커버리지 80% 목표 달성 어려움

개선:
- 파일 유형별 차등 기준
- 핵심 파일 90%, 일반 파일 70%

### 2. 워크플로우 패턴 개선
현재 문제:
- 간단한 버그도 QA 재현 단계 거침

개선:
- 명확한 버그는 재현 생략
- 바로 구현으로 이동

## Rule 수정 프로세스

1. 문제 식별
2. 개선안 작성
3. A/B 테스트 (10개 사이클)
4. 효과 측정
5. 채택 or 롤백

## 완료 기준
- [ ] Rule 수정 제안
- [ ] A/B 테스트 완료
- [ ] 개선 효과 10% 이상
