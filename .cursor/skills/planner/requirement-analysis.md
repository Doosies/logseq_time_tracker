# 요구사항 분석 가이드

이 Skill은 기획 에이전트가 사용자 요청을 분석하고 요구사항을 문서화하는 방법을 제공합니다.

## 요구사항 분석 프로세스

### 1. 사용자 요청 이해

```markdown
사용자 요청: "사용자 로그인 기능 추가해줘"

분석:
- 핵심 기능: 로그인
- 암시된 요구사항: 회원가입, 세션 관리, 로그아웃
- 불명확한 점: 인증 방식 (이메일/비밀번호? 소셜?)
```

### 2. 명확화 질문

불명확한 부분은 질문:

```markdown
질문 목록:
1. 인증 방식: 이메일/비밀번호? 소셜 로그인?
2. 세션 유지 기간: 얼마나?
3. 비밀번호 재설정 기능 필요?
4. 2단계 인증 (2FA) 필요?
```

### 3. 요구사항 추출

답변을 바탕으로 요구사항 작성

## 요구사항 유형

### 1. 기능 요구사항 (Functional Requirements)

시스템이 **무엇을 해야 하는지**

```markdown
## 기능 요구사항

### FR-1: 사용자 로그인
- 사용자는 이메일과 비밀번호로 로그인할 수 있다
- 로그인 성공 시 JWT 토큰을 발급한다
- 로그인 실패 시 명확한 에러 메시지를 표시한다

### FR-2: 사용자 회원가입
- 이메일, 비밀번호, 이름을 입력받는다
- 이메일 중복을 체크한다
- 비밀번호는 8자 이상, 영문/숫자/특수문자 포함이어야 한다

### FR-3: 비밀번호 재설정
- 이메일로 재설정 링크를 전송한다
- 링크는 1시간 동안 유효하다
```

### 2. 비기능 요구사항 (Non-Functional Requirements)

시스템의 **품질 속성**

```markdown
## 비기능 요구사항

### NFR-1: 성능
- 로그인 API 응답 시간: 200ms 이하
- 동시 접속자 1000명 처리 가능

### NFR-2: 보안
- 비밀번호는 bcrypt로 해싱 (10 rounds)
- JWT 토큰은 1시간 유효
- HTTPS 필수

### NFR-3: 사용성
- 로그인 폼은 3초 이내에 로드
- 에러 메시지는 사용자 친화적

### NFR-4: 신뢰성
- 시스템 가용성: 99.9%
- 자동 세션 갱신
```

## MoSCoW 우선순위

### Must have (필수)
```markdown
- 이메일/비밀번호 로그인
- 회원가입
- 로그아웃
- 세션 관리
```

### Should have (권장)
```markdown
- 비밀번호 재설정
- 이메일 검증
- 프로필 수정
```

### Could have (선택)
```markdown
- 소셜 로그인
- 2단계 인증
- 로그인 기록
```

### Won't have (제외)
```markdown
- 지문 인증
- 얼굴 인식
```

## 사용자 스토리

### 형식
```
As a [역할]
I want [기능]
So that [목적]
```

### 예시

```markdown
## 사용자 스토리

### US-1: 로그인
**As a** 사용자
**I want** 이메일과 비밀번호로 로그인하기
**So that** 내 계정에 접근할 수 있다

**수락 기준:**
- [ ] 올바른 이메일/비밀번호 입력 시 로그인 성공
- [ ] 잘못된 정보 입력 시 "이메일 또는 비밀번호가 잘못되었습니다" 표시
- [ ] 로그인 성공 시 대시보드로 리다이렉트
- [ ] JWT 토큰이 쿠키에 저장됨

### US-2: 회원가입
**As a** 신규 사용자
**I want** 계정을 생성하기
**So that** 서비스를 사용할 수 있다

**수락 기준:**
- [ ] 이메일, 비밀번호, 이름 입력 필드 제공
- [ ] 이메일 중복 체크
- [ ] 비밀번호 강도 체크 (8자+, 영문/숫자/특수문자)
- [ ] 회원가입 성공 시 환영 이메일 전송
```

## 제약사항

### 기술적 제약
```markdown
- Node.js 18+
- PostgreSQL 14+
- JWT for 인증
- bcrypt for 비밀번호 해싱
```

### 비즈니스 제약
```markdown
- GDPR 준수 (유럽 사용자 데이터)
- 개인정보 보호법 준수
- 비밀번호는 평문 저장 금지
```

### 시간 제약
```markdown
- MVP: 2주 이내
- 전체 기능: 1개월 이내
```
