# 워크플로우 조율

## 실행 전략

### 순차 실행
의존성이 있는 서브태스크:
```
서브태스크 1 (User 모델) → 완료 대기
→ 서브태스크 2 (회원가입) → 완료 대기
→ 서브태스크 3 (로그인)
```

### 병렬 실행
독립적인 서브태스크:
```
서브태스크 2 (회원가입) ┐
                        ├→ 동시 실행
서브태스크 4 (로그아웃) ┘
```

## 실행 순서 결정

1. 의존성 그래프 생성
2. 병렬 가능 작업 식별
3. 크리티컬 패스 우선
4. 에이전트 호출 스케줄링
