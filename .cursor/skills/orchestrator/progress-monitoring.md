# 진행 상황 모니터링

## 추적 항목

### 서브태스크 레벨
```markdown
서브태스크 1: User 모델
- 상태: 완료 ✅
- 소요 시간: 18분
- 에이전트: 기획, 구현
- 품질: 통과

서브태스크 2: 회원가입
- 상태: 진행 중 🔄
- 진행률: 60% (구현 완료, 테스트 중)
- 소요 시간: 22분 (예상: 30분)
- 에이전트: 구현 (완료), QA (진행 중)
```

### 전체 진행률
```markdown
전체: 5개 중 1개 완료, 1개 진행 중
진행률: 20% (1/5) + 12% (0.6/5) = 32%
예상 완료: 45분 후
```

## 진행 상황 리포트

```markdown
# 진행 상황 리포트

**시작 시간**: 14:00
**현재 시간**: 14:40
**경과 시간**: 40분

## 완료된 서브태스크
✅ Task1: User 모델 (18분)

## 진행 중인 서브태스크
🔄 Task2: 회원가입 (22/30분, 73%)
   - 구현: 완료
   - QA: 진행 중 (테스트 7/10)

## 대기 중인 서브태스크
⏳ Task3: 로그인
⏳ Task4: 로그아웃
⏳ Task5: 토큰 갱신

## 예상 완료 시간
15:30 (50분 후)
```

## 문제 감지

### 지연 감지
```markdown
예상: 30분
실제: 45분 (150%)

원인 분석:
- QA 단계에서 5번 재시도
- 테스트 작성 시간 과소 평가

대응:
- 다음 유사 태스크 예상 시간 조정
```

### 블로킹 감지
```markdown
Task2가 30분 이상 진행 중
→ 다른 서브태스크 블로킹
→ 우선순위 조정 필요
```
