# 병렬 실행

## 병렬화 조건

### 독립성 체크
- [ ] 서로 다른 파일 수정
- [ ] 데이터 의존성 없음
- [ ] 리소스 충돌 없음

## 병렬 실행 패턴

### 패턴 1: 완전 독립
```markdown
회원가입 API  ┐
              ├→ 동시 실행
비밀번호 재설정 ┘
```

### 패턴 2: 부분 병렬
```markdown
Step 1: User 모델 (순차)
   ↓
Step 2: 회원가입 + 로그인 (병렬)
   ↓
Step 3: 통합 테스트 (순차)
```

## 병렬 실행 전략

```markdown
분석:
- 총 5개 서브태스크
- Task1 → 선행 필수
- Task2, Task3, Task4 → 독립적
- Task5 → Task3 완료 후

실행 계획:
1. Task1 실행 (순차)
2. Task2, Task3, Task4 동시 실행 (병렬)
3. Task5 실행 (순차, Task3 결과 사용)

예상 시간:
순차: 20 + 30 + 30 + 15 + 25 = 120분
병렬: 20 + max(30, 30, 15) + 25 = 75분
절감: 45분 (37.5%)
```

## 동기화 포인트

병렬 작업 후 결과 병합:
```markdown
Task2 완료 ┐
Task3 완료 ├→ 모두 대기 → 통합 테스트
Task4 완료 ┘
```
