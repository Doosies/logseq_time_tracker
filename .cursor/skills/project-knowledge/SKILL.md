# 프로젝트 지식 (Project Knowledge)

## 목적

이 스킬은 **이 프로젝트의 구체적 구현 사실**을 한 곳에 관리합니다.

## `project-conventions`과의 차이

| | `project-conventions` | `project-knowledge` |
|---|---|---|
| 관점 | **어떻게** 코드를 작성하는가 (HOW) | **무엇이** 구현되어 있는가 (WHAT) |
| 내용 | 네이밍 규칙, 파일 패턴, import 순서 | 스토어 초기화 순서, 함수 역할, 아키텍처 결정 |
| 변경 빈도 | 거의 변하지 않음 | 기능 추가/변경마다 업데이트 가능 |
| 예시 | "`snake_case` 변수명 사용" | "`initializeTheme()`은 FOUC 방지 2단계 중 비동기 단계" |

## 도메인 스킬과의 차이

도메인 스킬(`developer/references/*.md` 등)은 **다른 프로젝트에서도 재사용 가능한 범용 패턴**을 담습니다.
이 스킬은 해당 패턴이 **이 프로젝트에서 구체적으로 어떻게 구현되었는지**를 담습니다.

예시:
- 도메인 스킬: "FOUC 방지를 위해 동기+비동기 2단계 초기화 패턴을 사용하라"
- 프로젝트 지식: "`initializeThemeSync()`이 1단계, `initializeTheme()`이 2단계"

## 사용 시점

- 해당 패키지의 코드를 수정하거나 새 기능을 추가할 때
- 테스트를 작성할 때 기존 패턴을 확인할 때
- 마이그레이션이나 리팩토링 시 기존 구조를 파악할 때

## 참조 구조

패키지별로 reference 파일을 조직합니다:

```
references/
├── ecount-dev-tool.md    # Chrome Extension 패키지
├── uikit.md              # UI Kit 패키지 (필요 시 추가)
└── ...                   # 기타 패키지 (필요 시 추가)
```

## 유지보수 원칙

- 코드가 변경되면 이 문서도 함께 업데이트
- 구체적 함수명, 변수명, 초기화 순서 등 **사실 기반** 정보만 기록
- 원칙이나 패턴은 도메인 스킬에, 구현 사실은 여기에
