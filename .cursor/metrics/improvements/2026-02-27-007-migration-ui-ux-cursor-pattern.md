## 1. 버그 요약

| 항목 | 내용 |
|------|------|
| **태스크 유형** | bugfix |
| **증상** | SectionSettings 팝오버 내 `[data-drag-handle]`에 `cursor: grab/grabbing` 스타일 누락 |

### 2.2 규칙 갭 (Gap)

**문제점**: "인터랙션 패턴"은 언급되나, **구체적인 예시(체크리스트)**가 시각적 속성만 포함함.

| 구분 | 기존 체크리스트 예시 | 누락된 항목 |
|------|---------------------|-------------|
| 시각적 | 아이콘, 레이아웃, 그립 모양 | ✅ 포함 |
| **인터랙션 피드백** | (없음) | **cursor**, hover, transition, focus 스타일 |

- **"그립 모양"**: 그립 아이콘의 시각적 형태를 의미. `cursor: grab/grabbing`은 **인터랙션 피드백**으로, 별도 명시 필요
- **headless-components.md** Skill: `cursor: grab`, `cursor: pointer`를 "동작·접근성" 스타일로 분류 → 마이그레이션 시 보존 대상

### 3.2 에이전트 정의 파일 수정 제안 (직접 수정하지 않음)

#### 제안 1: developer.md - "라이브러리 마이그레이션 시" 섹션

**현재 (1번 항목)**:
```markdown
- 기존 시각적 모양(아이콘, 레이아웃, 그립 모양 등)을 문서화한 후 새 라이브러리로 **동일하게** 구현
```

**제안 (수정)**:
```markdown
- 기존 시각적 모양(아이콘, 레이아웃, 그립 모양 등)과 **인터랙션 피드백**(cursor: grab/grabbing, hover, transition, focus 스타일)을 문서화한 후 새 라이브러리로 **동일하게** 구현
- 기존 컴포넌트가 내장 스타일(예: `<Handle />`의 cursor)을 갖고 있었다면, 교체 요소에도 동일 스타일 적용
```

#### 제안 2: planner.md - "라이브러리 마이그레이션 설계 시" 체크리스트

**현재**:
```markdown
- [ ] **시각적 보존**: 기존 시각적 모양(아이콘, 레이아웃, 그립 모양 등)을 새 라이브러리로 **동일하게** 구현하는 방법 명시
```

**제안 (수정)**:
```markdown
- [ ] **시각적·인터랙션 보존**: 기존 시각적 모양(아이콘, 레이아웃, 그립 모양 등)과 인터랙션 피드백(cursor, hover, transition)을 새 라이브러리로 **동일하게** 구현하는 방법 명시
```

#### 제안 3: system-improvement.md - 교훈 기반 에러 패턴 (선택)

**현재**: "라이브러리 마이그레이션 시 UI/UX 회귀"로 이미 매핑됨

**제안 (보강)**: 구체적 하위 패턴 추가
```markdown
- 라이브러리 마이그레이션 시 UI/UX 회귀 → planner.md 마이그레이션 체크리스트, developer.md 마이그레이션 규칙
  - **하위 패턴**: 기존 컴포넌트 내장 스타일(cursor, hover 등)이 교체 요소에 미이전 시 → 인터랙션 피드백 체크리스트 보강
```
