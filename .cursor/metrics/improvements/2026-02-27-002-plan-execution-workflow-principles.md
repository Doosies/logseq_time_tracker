# plan-execution 워크플로우 원칙 추가

**일자**: 2026-02-27  
**유형**: 워크플로우 명확화

## 배경

plan-execution 명령 실행 시 6단계(커밋) 완료 후 7, 8, 9단계를 생략한 이슈 발생.

## 개선 내용

`.cursor/commands/plan-execution.md`에 다음 원칙 추가:

1. **단계 생략 금지**: 진입 시점부터 9단계까지 순차 실행
2. **중간 진입 가능**: 진입 가능 단계 1, 3, 5, 6, 7 명시

## 효과

- 에이전트가 6단계 이후 단계를 생략하는 실수 방지
- "커밋을 진행해줘" 등 단계별 요청 시 올바른 진입점 인식
