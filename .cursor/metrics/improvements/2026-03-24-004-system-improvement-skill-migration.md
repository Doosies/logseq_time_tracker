## 변경 내용

system-improvement를 서브에이전트에서 메인 에이전트 직접 수행 스킬로 전환.

## 변경 이유

- 시스템 개선은 메인 에이전트의 핵심 책임 중 하나
- 서브에이전트 위임 시 컨텍스트 전달 오버헤드 발생
- 메인 에이전트가 세션 전체 맥락을 보유하므로 직접 분석이 더 효과적

## 예상 효과

- 서브에이전트 호출 1회 감소 (system-improvement Task 호출 제거)
- 메인 에이전트의 세션 컨텍스트를 직접 활용하여 더 정확한 패턴 분석
- 에이전트 수 8개 → 7개로 단순화
