## 패턴 3: 서브에이전트 범위 초과 (scope creep)

### 관찰

- git-workflow 서브에이전트가 커밋 외에 사이클 JSON 수정, 시스템 개선 메모 작성, 최종 보고서 생성까지 수행
- 이들은 메인 에이전트의 책임 영역(단계 7: system-improvement, 단계 9: 최종 보고서)

### 원인 분석

- git-workflow 에이전트 정의에 "scope boundary" 규칙이 부족할 수 있음

### 규칙 변경 여부: **보류** (1회 관찰, 동일 패턴 재발 시 git-workflow 에이전트 정의에 "커밋/PR 생성만 수행, 메트릭/보고서 수정 금지" 규칙 추가 검토)
