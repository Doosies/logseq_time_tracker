# 시스템 개선 리포트: 테스트 ID 기반 워크플로우

**사이클**: 2026-03-25-013
**유형**: 규칙/스킬 최적화
**일시**: 2026-03-25

## 개선 내용

### 문제
- 테스트 코드와 유즈케이스 명세 간 추적성 부재
- 테스트 스펙이 1305줄 단일 파일에 집중되어 접근성/유지보수성 저하
- 테스트 누락/중복 감지 어려움

### 변경 사항

#### 1. QA 에이전트 정의 (`.cursor/agents/qa.md`)
- **2단계 테스트 전략 수립**에 "테스트 ID 기반 작성 (필수)" 섹션 추가
  - `__test_specs__/{level}/{domain}.md`에서 ID 확인 워크플로우
  - SSOT 정의 (패키지별 `__test_specs__/` + Phase 인덱스)
- **3단계 테스트 작성**에 "테스트 ID 추적성" 항목 추가
- **품질 게이트 > 기본 요구사항**에 "UC-* ID 포함" 체크 항목 추가

#### 2. 테스트 품질 스킬 (`.cursor/skills/qa/references/test-quality.md`)
- 가독성 체크리스트에 UC-* ID 필수 항목 추가
- 좋은 예시를 ID 포함 형태로 업데이트
- 코드 리뷰 체크리스트에 ID 확인 추가

#### 3. 테스트 전략 스킬 (`.cursor/skills/qa/references/test-strategy.md`)
- "Test ID 기반 워크플로우" 섹션 신규 추가
  - ID 체계 테이블
  - 스펙 먼저 → ID → 코드 워크플로우
  - SSOT 정의
  - 좋은/나쁜 예시

### 기대 효과
- 모든 테스트가 유즈케이스 명세와 1:1 매핑 → 추적성 확보
- 패키지별 테스트 스펙이 코드 바로 옆에 위치 → 접근성 향상
- 테스트 누락/중복 즉시 감지 가능
- QA 에이전트가 자동으로 ID 기반 워크플로우를 따름
