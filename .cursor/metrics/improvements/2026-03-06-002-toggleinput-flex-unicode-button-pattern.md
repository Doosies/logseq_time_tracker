## 2. 발견된 패턴/이슈

### 2.1 CSS flex layout에서 아이콘 버튼 축소 (핵심)

| 구분 | 내용 |
|------|------|
| **증상** | flex 컨테이너 내 아이콘 버튼이 0 너비로 축소되어 시각적으로 사라짐 |
| **원인** | `flexShrink: 0` 누락. flex 기본값 `flex-shrink: 1`로 공간 부족 시 축소됨 |
| **해결** | 아이콘/버튼 요소에 `flexShrink: 0` 또는 `flex-shrink: 0` 적용 |
| **재발 가능성** | **높음** - improvements 폴더에 CSS 레이아웃 이슈 다수 (2026-02-27-001, 005, 006) |

**교훈**: flex 컨테이너 내 **고정 크기 요소**(아이콘, 버튼, 이미지)는 `flex-shrink: 0`을 명시하여 축소 방지.

### 2.2 유니코드 문자 렌더링 호환성

| 구분 | 내용 |
|------|------|
| **증상** | `⇄` (U+21C4) 문자가 시스템 폰트에서 렌더링되지 않음 |
| **해결** | SVG 아이콘으로 교체 |
| **재발 가능성** | **낮음** - 특수 유니코드 사용 시에만 발생 |

**교훈**: 교환/스위칭 등 특수 기호는 **시스템 폰트 호환성**을 고려하여 SVG 아이콘 사용 권장.

### 2.3 브라우저 기본 버튼 스타일 미리셋

| 구분 | 내용 |
|------|------|
| **증상** | `<button>` 요소에 CSS 리셋 없이 기본 3D 테두리/배경 적용 |
| **해결** | `appearance: none`, `border: none`, `background: none` 등 리셋 적용 |
| **재발 가능성** | **중간** - 새 버튼 추가 시 누락 가능 |

**교훈**: headless-components.md에 Reset 스타일 언급 있으나, **버튼 기본 스타일 리셋**은 프로젝트 전역 규칙으로 명시되지 않음.

### 2.4 QA 중 관련 없는 테스트 실패 발견

| 구분 | 내용 |
|------|------|
| **증상** | calculator 섹션 추가 후 관련 테스트/스냅샷 미갱신 → QA 검증 시 실패 |
| **해결** | QA가 직접 수정 (Proactive 수정) |
| **재발 가능성** | **중간** - 구조 변경 시 관련 테스트 갱신 누락 패턴 |

**교훈**: 2026-02-27-001 "QA Proactive 수정" 규칙 적용됨. 2026-03-06-001 "마이그레이션 테스트 보강"과는 별개(섹션 추가는 마이그레이션 아님).

## 4. 에이전트 규칙 개선 여부

### 4.1 flexShrink 패턴 → **규칙/Skill 추가 권장 (P2)**

| 근거 | 설명 |
|------|------|
| **반복 패턴** | CSS layout 이슈 4건 누적. flexShrink는 구체적이고 검증 가능한 패턴 |
| **낮은 부담** | 체크리스트 1줄 추가 수준 (`flex 컨테이너 내 고정 크기 요소 → flex-shrink: 0`) |
| **재발 가능성** | flex + 아이콘/버튼 조합은 UI에서 빈번 |

**제안**: `developer/svelte-conventions.md` 또는 `developer` Skill에 "CSS flex layout" 체크리스트 추가.

```markdown
### Flex 컨테이너 내 고정 크기 요소 (권장)

flex 레이아웃에서 아이콘, 버튼, 이미지 등 **고정 크기 요소**가 축소되지 않도록:

- `flex-shrink: 0` 또는 `flexShrink: 0` 명시
- 기본값 `flex-shrink: 1`로 공간 부족 시 0 너비까지 축소될 수 있음
```

## 5. 개선 제안 (flexShrink만)

### 5.1 developer Skill - CSS flex 체크리스트 추가

**추가 위치**: `developer/references/svelte-conventions.md` 또는 신규 `developer/references/css-layout-patterns.md`

**제안 내용**:

```markdown
### Flex 레이아웃 - 고정 크기 요소 축소 방지

flex 컨테이너(`display: flex`) 내부에 아이콘, 버튼, 이미지 등 **고정 크기 요소**가 있을 때:

- **문제**: `flex-shrink` 기본값 1로, 공간 부족 시 0 너비까지 축소 가능
- **해결**: 해당 요소에 `flex-shrink: 0` 또는 `flexShrink: 0` 명시
- **체크**: flex 레이아웃에서 "보이지 않는" 버튼/아이콘 이슈 시 우선 확인
```

### 5.2 system-improvement.md - 교훈 패턴 매핑 (선택)

**추가 위치**: "교훈 기반 에러 패턴 분석" (2단계 병목 분석)

```markdown
- flex 컨테이너 내 아이콘/버튼 축소 (flexShrink 누락) → developer.md/svelte-conventions.md CSS flex 체크리스트
```
