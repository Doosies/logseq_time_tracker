# 시스템 개선 분석: 테스트 범위 검증 누락 패턴

**사이클**: 2026-03-25-005
**태스크**: Phase 3 누락 컴포넌트 테스트(UC-UI-009~018) 및 E2E 테스트(UC-E2E-003) 보완
**분석일**: 2026-03-25

---

## 1. 문제 요약

Phase 3G 테스트 단계에서 컴포넌트 테스트 10건과 E2E 테스트 1건이 누락됨. 사용자가 직접 발견하여 지적한 후에야 보완 작업 진행.

### 근본 원인 (Root Cause)

메인 에이전트가 **가정 기반 판단**을 함:
1. `@testing-library/svelte`가 설치되지 않았다고 **가정** → 컴포넌트 테스트 스킵 지시
2. Playwright E2E 설정이 없다고 **가정** → E2E 테스트 스킵 지시

실제로는 둘 다 설치되어 있었고, 기존 테스트 파일도 존재했음.

---

## 2. 병목 지점 식별

### 병목 1: 테스트 인프라 사전 검증 부재 (Impact: Critical)

**현상**: QA에게 테스트 작성을 위임하기 전에 테스트 인프라를 검증하지 않음
**영향**: 전체 테스트 범위의 ~30% 누락 (11/~40건)
**빈도**: 이번 사이클에서 최초 발견이지만, 구조적 취약점

### 병목 2: 테스트 완료 후 범위 교차 검증 부재 (Impact: High)

**현상**: QA가 테스트를 완료했다고 보고 후, 메인 에이전트가 테스트 계획(3g-tests.md)과 실제 작성된 테스트를 대조하지 않음
**영향**: 누락을 발견하지 못하고 "완료"로 판단

### 병목 3: E2E 전제조건 미식별 (Impact: Medium)

**현상**: UC-E2E-003의 Given("풀화면 UI가 열려 있다")이 App.svelte 수정을 필요로 하는데, Phase 3 계획에서 이를 명시적 전제조건으로 식별하지 못함
**영향**: E2E 작성 시 추가 구현 작업 필요 (App.svelte FullView 통합)

## 3. 개선안

### 개선안 1: QA 위임 전 인프라 검증 체크리스트 추가

**적용 대상**: `main-orchestrator.mdc` > QA 단계 전 체크포인트

**변경 내용**:
```markdown
### QA 위임 전 필수 확인 (신규)
QA 서브에이전트에게 테스트 작성을 위임하기 전:
1. **package.json devDependencies 확인**: 테스트 관련 패키지 존재 여부
   - `vitest`, `@testing-library/svelte`, `playwright` 등
2. **기존 테스트 파일 확인**: 1-2개 기존 테스트 파일 Read로 패턴 확인
3. **테스트 계획 문서 전체 UC-ID 목록화**: grep으로 UC- 패턴 추출
```

**예상 효과**: 인프라 부재 가정으로 인한 테스트 누락 방지

### 개선안 2: 테스트 완료 후 범위 교차 검증 추가

**적용 대상**: `main-orchestrator.mdc` > QA 단계 후 품질 게이트

**변경 내용**:
```markdown
### QA 단계 후 (기존 체크리스트에 추가)
- [ ] **테스트 범위 교차 검증** (필수!): 테스트 계획 문서의 UC-ID 목록과 
      실제 생성된 테스트 파일의 UC-ID를 대조. 누락된 UC-ID가 있으면 QA에 재요청.
      검증 방법: `grep -r "UC-" --include="*.test.ts" <test-dir>` 결과와 계획 비교
```

**예상 효과**: QA 완료 보고 후 누락 자동 감지

### 개선안 3: Planner 테스트 계획에 전제조건 명시 의무화

**적용 대상**: `planner.md` > 테스트 계획 작성 규칙

**변경 내용**:
```markdown
### E2E 테스트 계획 시 필수 항목
- Given 조건에 UI 상태가 포함된 경우, 해당 UI 상태를 만들기 위한 **구현 전제조건** 명시
- 예: "풀화면 UI가 열려 있다" → "App.svelte에 FullView 통합 필요 (구현 전제)"
```

**예상 효과**: E2E 테스트의 구현 의존성 사전 식별
