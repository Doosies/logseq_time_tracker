## TODO 필수 형식

### 기본 구조
```yaml
todos:
  - id: task-id
    content: "[병렬-N|직렬-N] 작업 설명 (담당: agent-name, 선행: task-id)"
    status: pending
```

### 필수 요소
- **실행 순서**: `[병렬-N]` 또는 `[직렬-N]` (동일 N = 병렬, 다른 N = 직렬-낮은 N 우선)
- **담당 서브에이전트**: developer, qa, docs, security, planner, git-workflow, system-improvement
- **선행 조건**: 의존 관계 있을 시 `선행: task-id` 명시

### 예시

#### Chore: 스크립트 실행 시점 기본값 설정
```yaml
todos:
  - id: hide-run-at-ui
    content: "[병렬-1] ScriptEditor UI 숨기기 (담당: developer)"
  - id: update-migration-logic
    content: "[병렬-1] user_scripts 마이그레이션 로직 수정 (담당: developer)"
  - id: update-tests
    content: "[직렬-2] 테스트 파일 업데이트 (담당: qa, 선행: hide-run-at-ui, update-migration-logic)"
  - id: update-docs
    content: "[직렬-3] 문서 업데이트 (담당: docs, 선행: update-tests)"
```

#### Feature: 사용자 인증
```yaml
todos:
  - id: design
    content: "[직렬-1] 인증 API 설계 (담당: planner)"
  - id: impl-register
    content: "[병렬-1] 회원가입 API 구현 (담당: developer, 선행: design)"
  - id: impl-login
    content: "[병렬-2] 로그인 API 구현 (담당: developer, 선행: design)"
  - id: test-auth
    content: "[직렬-2] 인증 통합 테스트 (담당: qa, 선행: impl-register, impl-login)"
```
